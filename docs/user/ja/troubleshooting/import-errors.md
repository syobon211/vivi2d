# 読み込みエラー

## 翻訳ドラフトについて

このページは翻訳ドラフトです。最新かつ最も詳しい手順は、同じトピックの英語版に先に反映されます。公開前に、この翻訳も内容を広げてレビューします。

問題がファイル形式、サイズ、アクセス権、または予期しない元データにあるのかを切り分けます。

## 用意するもの

- 開けなかったファイル。
- 小さくて正常に開けることが分かっているテストファイル。
- 一時ダウンロード場所ではない安全なプロジェクトフォルダー。

## 手順

1. まず小さな正常ファイルを開いてみます。
2. 拡張子と実際のファイル内容が一致しているか確認します。
3. テスト用コピーを通常のプロジェクトフォルダーへ移動して再試行します。
4. 元ファイルが大きい場合は、小さいコピーでワークフローを確認します。
5. 1つの素材だけ失敗する場合は、元ファイルを変えずにコピーで作業します。

## 結果を確認する

正常なファイルが開けるなら、問題は失敗した素材、またはその保存場所にある可能性が高いです。

## うまくいかないとき

公開レポートにローカルのスタックトレース、非公開ファイル名、完全なファイルパスを貼らないでください。ファイル形式、おおよそのサイズ、失敗した画面上の操作を説明します。
