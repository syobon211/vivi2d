# Viewer API の基本

## 翻訳ドラフトについて

このページは翻訳ドラフトです。最新かつ最も詳しい手順は、同じトピックの英語版に先に反映されます。公開前に、この翻訳も内容を広げてレビューします。

ローカルツールから Viewer の状態を操作、または確認したいときに Viewer API を使います。

## 用意するもの

- ローカルで起動している Viewer。
- 公開 Viewer API クライアントの流れを使うクライアント。
- Viewer セッションとペアリングする権限。

## 手順

1. Viewer を開き、ペアリング中は見える状態にしておきます。
2. Viewer に接続するブラウザサンプル、またはツールを起動します。
3. ペアリング前に、接続先がローカルで想定どおりであることを確認します。
4. Viewer またはクライアントに表示されるペアリング確認を完了します。
5. 制御を許可する前に、要求されるスコープを確認します。
6. 制御コマンドを送る前に、状態取得など小さく安全なリクエストを送ります。
7. テストが終わったら、セッションを取り消すか閉じます。

## 結果を確認する

クライアントがペアリング済み、または認証済みの状態を表示し、Viewer は許可されたスコープ内のリクエストにだけ反応します。

## うまくいかないとき

トークン、秘密を含む接続先 URL、または生の通信内容を公開レポートに貼らないでください。見えている状態と失敗した操作だけを説明します。
