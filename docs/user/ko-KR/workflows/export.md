# 내보내기

## 번역 초안 안내

이 페이지는 번역 초안입니다. 가장 최신이고 자세한 절차는 같은 주제의 영어판에 먼저 반영됩니다. 공개 전에는 이 번역도 내용을 확장하고 다시 검토합니다.

편집기와 Viewer에서 프로젝트를 확인한 뒤 내보냅니다.

## 필요한 것

- 차단 검토 경고가 없는 저장된 프로젝트.
- 원본 아트워크를 덮어쓰지 않는 대상 폴더.
- 미리보기, 공유, 연동 테스트 같은 대상 워크플로.

## 단계

1. 내보내기 컨트롤을 열기 전에 프로젝트를 저장합니다.
2. 테스트하려는 워크플로에 맞는 내보내기 대상을 선택합니다.
3. 앱에 표시되는 실험적 기능이나 호환성 안내를 읽습니다.
4. 결과를 확인하기 쉽도록 새 폴더로 내보냅니다.
5. 공유하기 전에 예상 Viewer나 샘플에서 내보낸 결과를 엽니다.

## 결과 확인

내보낸 파일이 완전하고 식별하기 쉬우며, 비공개 작업 폴더의 파일 없이도 대상 워크플로에서 불러와져야 합니다.

## 문제가 있을 때

텍스처, 레이어, 움직임이 빠졌다면 프로젝트로 돌아가 원본을 확인한 뒤 다시 내보냅니다. 비교를 위해 정상 내보내기 결과 하나를 보관하면 좋습니다.
