# Viewer API 기본

## 번역 초안 안내

이 페이지는 번역 초안입니다. 가장 최신이고 자세한 절차는 같은 주제의 영어판에 먼저 반영됩니다. 공개 전에는 이 번역도 내용을 확장하고 다시 검토합니다.

로컬 도구가 Viewer 상태를 제어하거나 관찰해야 할 때 Viewer API를 사용합니다.

## 필요한 것

- 로컬에서 실행 중인 Viewer.
- 공개 Viewer API 클라이언트 흐름을 사용하는 클라이언트.
- Viewer 세션과 페어링할 권한.

## 단계

1. Viewer를 열고 페어링 중에는 보이도록 둡니다.
2. Viewer에 연결하는 브라우저 샘플이나 도구를 시작합니다.
3. 페어링 전에 엔드포인트가 로컬이고 예상한 대상인지 확인합니다.
4. Viewer 또는 클라이언트에 표시된 페어링 확인을 완료합니다.
5. 제어를 허용하기 전에 요청된 범위를 확인합니다.
6. 제어 명령을 보내기 전에 상태 읽기처럼 작고 안전한 요청을 먼저 보냅니다.
7. 테스트가 끝나면 세션을 취소하거나 닫습니다.

## 결과 확인

클라이언트는 페어링 또는 인증 상태를 보여야 하며, Viewer는 허용된 범위에 포함된 요청에만 반응해야 합니다.

## 문제가 있을 때

토큰, 비밀이 포함된 엔드포인트 URL, 원시 통신 내용을 공개 보고서에 붙여 넣지 마세요. 보이는 상태와 실패한 동작만 설명합니다.
