# Internationalization Task Guide

## When To Use This Guide

Use this guide when adding or changing UI strings, locale normalization, locale
storage behavior, translation review metadata, or locale E2E coverage.

## Ownership And Boundaries

- Supported UI locales are `en`, `ja`, `zh-Hans`, and `ko-KR`.
- URL or test locale overrides must not overwrite persisted user preferences.
- `zh-Hans` means Simplified Chinese only. It must not imply Traditional
  Chinese support.
- No visible UI should ship with mojibake, tofu glyphs, hidden English fallback,
  or stale translation-review metadata.
- Machine-assisted draft copy requires current human review metadata before it
  ships.

## Primary Files

- `src/lib/i18n/**`
- `packages/viewer/src/i18n/**`
- `docs/developer/contributing/i18n/translation-review-manifest.json`
- `scripts/check-i18n-*.mjs`
- `e2e/specs/i18n-*.spec.ts`

## Tests And Gates

```bash
npm run i18n:check:all
npm run check:i18n-review-manifest
npm run test:e2e:smoke
npm run check:docs-public-surface
```

Run a focused locale E2E or screenshot pass when the changed copy is visible in
Editor, Viewer, dialogs, or menus. Run `npm run check:docs-public-surface` when
docs or screenshots change.

## Safe Change Pattern

1. Add the English source string first.
2. Add matching keys in `ja`, `zh-Hans`, and `ko-KR`.
3. Update review metadata for changed namespaces.
4. Check layout-sensitive screens in the longest affected locale.
5. Avoid locale-specific behavior outside the shared normalization helper.

## Common Failure Modes

- A new key exists in English but falls back in another locale.
- A URL/test override persists into user settings.
- Locale tags with underscores, casing differences, or extensions normalize
  differently between Editor and Viewer.
- Screenshot or workflow text is translated but the review manifest is stale.
- User docs and UI copy diverge on supported locale claims.

## PR Checklist

- All supported locales have the same key set.
- Review manifest entries are current for changed namespaces.
- Layout and overflow checks cover the changed surface.
- Locale fallback behavior is deterministic.
- Public docs avoid unsupported language-support claims.
