# Code of Conduct

Vivi2D aims to be a welcoming project for artists, tool builders, runtime
developers, researchers, and maintainers. We want technical discussions to be
honest, specific, and safe enough that people can learn in public.

This is a Vivi2D-specific conduct policy, not a verbatim adoption of an external
template.

## Our Standards

Expected behavior:

- Be respectful, constructive, and patient.
- Assume good intent when possible, and accept corrections gracefully.
- Keep critique focused on code, design, tests, documentation, user impact, and
  project goals.
- Credit prior work, research, examples, and third-party sources clearly.
- Respect boundaries around private assets, unpublished work, security reports,
  personal information, and maintainer time.
- Make room for people with different skill levels, languages, tools, and
  backgrounds.

Unacceptable behavior:

- Harassment, threats, stalking, or sustained disruption.
- Discriminatory language or conduct related to identity, background,
  disability, age, nationality, race, religion, gender, sexual orientation, or
  similar protected characteristics.
- Sexualized language or imagery in project spaces.
- Publishing private messages, personal information, credentials, unreleased
  assets, private model files, or security details without permission.
- Repeatedly derailing technical discussions after maintainers ask to refocus.
- Misrepresenting affiliation, licensing, provenance, review status, or legal
  approval.
- Pressuring others to disclose private information, private assets, or
  unpublished work.

## Scope

This code applies in project spaces such as issues, pull requests, Discussions,
GitHub Actions comments, release notes, community chat spaces controlled by the
project, and public project-related interactions.

It also applies when behavior outside project spaces directly affects the safety
or maintainability of the project.

## Reporting

If you see behavior that violates this code, use the private conduct contact
published in the repository profile or organization profile. If a dedicated
conduct contact is not available yet, avoid posting sensitive details publicly
and use GitHub's platform reporting tools for urgent safety concerns.

If the report involves a security vulnerability, use the private process in
`SECURITY.md` instead.

Please include:

- what happened
- where it happened
- links, screenshots, or context when safe to share
- whether immediate moderation is needed

Do not post harassment reports or vulnerability details in public issues.

## Enforcement

Maintainers may take actions that are appropriate to the situation, including:

- clarifying expectations or redirecting a discussion
- editing or deleting comments that expose private information or unsafe details
- closing, locking, or limiting participation in issues or pull requests
- requesting a temporary pause from project spaces
- blocking users from project spaces
- declining contributions that violate project safety, licensing, provenance, or
  security requirements

Maintainers should apply this code consistently and avoid using it to suppress
good-faith technical disagreement. Healthy disagreement is welcome when it stays
respectful, specific, and focused on the work.
